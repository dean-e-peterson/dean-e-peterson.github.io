How the Web Works

The World Wide Web is a decentralized system for storing, referencing, and retrieving information. The term Internet is often used colloquially as a synonym for the World Wide Web, but at a technical level they are separate. The internet is the global network of computers which provides the connections that the web uses to retrieve information.

Storing Information

The data accessible via the World Wide Web is stored on web servers all over the world. These are computers whose purpose is to store web pages and other files, and to pass them over the network upon request. Anyone can run their own web server, although they may have to pay an internet service provider or telecommunications company to provide the connection to the rest of the internet. Most individuals and small companies rent space on a web hosting company's web servers for a fee.

Referencing Information

The web's native form of content is the web page, a text document with special markup (HTML) indicating document structure and containing hyperlinks, which are links to any other document on the Web. The simplicity and ubiquity of hyperlinks has profound implications, allowing any document to reference any other (public) document in the world, creating the "web" of references that gives the World Wide Web its name.

Web pages are displayed by computer programs called web browsers. These are the windows to the web which run on computers, smartphones, etc. that people look at to view the web. Browsers format the web pages for viewing, and allow the user to select the hyperlinks to navigate between documents on the web.

Retrieving Information

When a user enters a web address or clicks on a hyperlink, behind the scenes, the web browser uses the global DNS database to convert the web server's domain name into a numeric IP address. Then, the browser talks to the web server over the internet to request the desired document. The request is formatted in a specific way, per the HTTP protocol. Upon receiving the request, the web server sends the requested web page back over the internet to the browser that requested it.

CSS Specification Musings

I wonder if CSS specification clarity and organization is a casualty of the tension between the W3C and browser implementers.

The latest CSS 2 revision has a red note saying that “several sections ... have been updated by other specifications”, and points you to a portion of a document that has indexes into those specifications. In other words, there is no single official CSS 3 specification document, as I understand it. I'm sure I don't know the challenges they face trying to standardize a moving target, but it does seem a bit unfortunate.

The Decentralized Nature of the Web

The decentralized nature of the World Wide Web is a key strength on both technological and social levels. Technologically, it makes it harder to destroy any significant proportion of the internet. Socially, the World Wide Web represents the greatest leap in the publication and availability of information since the printing press.

The web's decentralized nature does have some drawbacks, as criminals or other bad actors can more easily sell illicit goods from hiding. Also, there is no common provision of backup and continued access to information in the event of a catastrophe.

The decentralization of the web is somewhat limited by the prevalence of large telecommunications companies providing the backbone network links that carry much of the traffic, and by the influence of search engines, especially Google, as the way most people find content on the web in the first place.

Steps in a Web Request

Step Number Description
1 User clicks link in web page displayed in web browser.
2 Web browser queries DNS to convert destination domain name to an IP address.
3 Web browser requests the destination web page from the web server at the IP address returned by DNS.
4 Web server responds, sending back the destination web page.
5 Web browser receives the destination web page and displays it.
6 User reads the new web page, until they click on a link and the steps repeat from step 1.
